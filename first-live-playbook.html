The set up

Gear, room,
software stack.

Three parts of the physical and digital environment that need to be locked before stream day. None are optional.

①
Gear (what we actually have)
Laptop · Loom · cheap audio
  • Camera: built-in laptop webcam
  • Position laptop at eye level (stack books or laptop stand)
  • Audio: AirPods or cheap USB mic ($30 Fifine works)
  • Built-in laptop mic = last resort, sounds tinny
  • Light: face a window OR desk lamp behind laptop
  • No overhead lights only — kills your face
②
Room
Background · sound · network
  • Clean wall OR Videfy virtual background in Meet/Discord
  • No echoey rooms — pick a small carpeted space
  • Close door · kill HVAC noise · pets out
  • Hardwired ethernet if possible · phone hotspot fallback
  • All notifications OFF (Slack, iMessage, email)
  • Phone on silent, face down
③
Software stack
Live + record + chat
  • Live: Discord Stage (or Google Meet) — native screen share
  • Record: Loom running in background, full-screen capture
  • Loom = backup recording AND repurpose source
  • Videfy platform open in 1 browser tab, full-screen for share
  • Notes/outline on phone or 2nd device — never the streaming machine
  • CTA word STUDIO dropped in chat at minute 5 / 25 / close

Things to lock
before going live.

Pick the project to walk through · Vaughn + Alex
Real client work or demo brand. Vaughn drives the choice since he's running the walkthrough. Alex confirms it ladders to the live's title + audience promise.

Pre-write 5-8 questions in case chat is quiet. Mix tactical ("how do you handle revisions?") with strategic ("where does AI hit its limit?"). Routing notes per question (who answers).

Channels: #announcements, #live-stage, #project-feedback, #beta-bugs. Roles: @host, @team, @member. Invite link ready to paste anywhere. Owner not yet assigned — see Live Ops options.
6
Run a private test session · Full team
All 4 in Discord. Test cam, audio, screen share, Loom recording. Watch Loom playback together. Fix what breaks before announcing publicly.
7
Announce + collect signups · Alex (post) + Live Ops (Discord pin)
Post on LinkedIn (company), X, IG. Email beta list. Discord invite link in every post. Pin announcement in server.
8
Reminder pass · Alex
24 hours before: reminder email + story countdown + repost on each channel. Confirm all 4 team members are good to go.
9
Day-of buffer · 1 hour before · Full team
Set rooms. Test webcam + audio + screen share in Discord. Live Ops confirms Loom records the right screen. Vaughn opens Videfy tab + logs in. Everyone joins the room 5 min early with cam off.
10
Pre-flight checklist · Live Ops runs · whole team confirms
Webcam on · Audio working · Screen share = Videfy tab · Loom recording · Notifications OFF · Phone silent · Outline visible on 2nd device. Live Ops calls each item out loud, each person confirms theirs.
Who does what

Four people.
Four lanes.

Each role has one job. No overlap, no improvising into someone else's lane. The walkthrough host drives the build. The technical answers tech. The domain expert answers strategy. The host runs flow + Q&A pacing.

①
Vaughn — walkthrough lead
Drives the build narration
  • Shares screen, walks through the Videfy project end-to-end
  • Narrates what he's clicking and why, in real time
  • Owns pacing of the build segment (20-30 min)
  • Hands off to Alex for transitions + Q&A
  • Stays focused on the screen share, not the chat
②
Marta — platform technical (conditional)
Only joins if live is platform-focused
  • Joins only when live showcases platform tech
  • Answers deep technical questions about the Videfy platform
  • Backs up Vaughn during walkthrough · jumps in if a feature breaks
  • Catches platform bugs live · notes for engineering
  • Speaks to roadmap + what's shipping next
  • Not responsible for Discord, Loom, or live ops tech
③
Doron — domain knowledge
Industry + strategy depth
  • Answers deeper questions on video production workflow
  • Frames AI tooling context for the audience
  • Speaks to "why this matters" for agencies + creators
  • Steps in during Q&A when domain expertise needed
  • Adds anecdotes + war stories for credibility
④
Alex — host / moderator
Flow + Q&A + CTA
  • Opens the live · sets context · introduces the team
  • Manages transitions between segments
  • Reads chat questions aloud · routes to right person
  • Calls CTA at minute 5 / 25 / close
  • Owns the clock · wraps cleanly at 60 min
Routing rule for Q&A
Alex reads the question. Decides routing on the fly: "How does X feature work?" → Vaughn or Marta (if platform live) · "Why does video production work this way?" → Doron · "What's the roadmap?" → Alex. Anyone can pass to anyone. No question lingers on the wrong person.
Open role · Live Ops · owner TBD
Marta is platform-technical, not live-ops-technical. We still need someone to own Discord server setup, Loom recording, audience monitoring, and tech troubleshooting during the stream. Options to consider:
  • Alex absorbs it — keeps team at 4. Adds load on host (already running outline + Q&A + clock). Risk: spread thin live.
  • Doron absorbs it — currently lightest live load (Q&A only). Could double-hat as Live Ops + domain expert. Recommended if he has bandwidth.
  • Hire a part-time producer / VA — $15-30/hr, remote. One-time Discord setup + monthly live ops. Cleanest separation of duties.
  • Rotate — different team member each live until role is hired.

Options · platform

Where to host
when no accounts exist yet.

No X Live · no YouTube channel · no LinkedIn Live approval. We're spinning something up cold. Comparison weights setup time, replay value, and whether the platform leaves a community asset behind after the stream ends.

PlatformSetup timeProsCons
Discord Stage~1 hourFree · server doubles as permanent Videfy community · audience returns between lives · screen share clean · dev/creator crowd already hereNeed to build a server from scratch · audience must join server first
Google Meet~5 minFastest start · everyone has a Google account · no install · screen share worksNo replay community · 100-person cap free tier · 60-min cap on free · throwaway after stream
Zoom Webinar~30 minPolished UX · registration capture · replay built-in · Q&A panel · best-in-class screen sharePaid only ($79+/mo) · feels corporate · no community asset after
StreamYard~20 minBrowser-based · streams to X / YouTube / LinkedIn / custom RTMP · free tier works · branded overlaysStill need a destination platform · free tier watermarked
Riverside.fm~20 minStudio-quality recording · separate audio/video tracks · live + post-edit readyPaid for full features · less of a "live" feel · audience as guests, not viewers
Twitch~30 minFree · replay built-in · chat native · low barrierGaming-coded · audience tiny without warm-up · weak B2B fit
Recommend — Discord Stage primary
Discord wins because the server itself is the asset. Spin one up once, every future live keeps the audience there, and it compounds into a community where Videfy beta users hang out, share builds, and ask questions between streams. Google Meet ships fastest but leaves nothing behind. StreamYard is the bridge layer to add later once X/YouTube accounts warm up.
Account setup path · 3 steps · ~90 min total
1. Create Discord server (free) — channels: #announcements, #live-stage, #project-feedback, #beta-bugs · invite link in email + bio. 2. Set up Stage channel for live walkthroughs · use Discord's native screen share (no OBS needed for rough cut). 3. Optional add-on later: connect StreamYard to broadcast Discord audio/video into X + YouTube simultaneously once those channels grow.
Options · format

Four formats.
Pick one for the first.

All four work. Each weights production effort and audience engagement differently. For live #1 we want the lowest variance, not the highest ceiling.

A
Pure walkthrough
Recommended · live #1
Full project end-to-end. Q&A at the end, one chunk. Cleanest, best replay. Lowest tech variance.
B
Build-along
Audience-driven
Audience suggests prompts/edits live. Higher engagement, harder to control. Best for live #3+.
C
Guest + demo
Reach amplifier
Invite a client or partner. Doubles reach if guest has audience. Adds a coordination dependency.
D
Office hours
Recurring cadence
Open Q&A only. No scripted walkthrough. Use as the recurring slot once first live lands.

Field 05 · What the build showcases

Two layers must come through every live: the platform features in action, AND the Videfy methodology behind why it works that way.

▸
Platform features shown
The "what"
  • Project workspace · brief intake · brand context
  • Asset organization + memory across the project
  • AI generation surfaces (Claude MCP · Avatar · Generate)
  • Scene-level editing built in
  • Client review · no paid seat required
  • Collaborator invite + permission model
  • Export + handoff package
▸
Videfy methodology
The "why"
  • Idea → Feedback → Finished as one continuous flow
  • Production system absorbs the 6-tool stack into one tab
  • Memory holds the project arc · no zero-state every time
  • AI activates inside the project, not as a separate generator
  • Client review is free · removes the seat tax
  • Collaboration without ramp time
  • Production without the chaos
Rule
Every live must hit at least 3 platform features and 2 methodology beats. If it doesn't, the live is a demo, not a story. The story is what converts.
