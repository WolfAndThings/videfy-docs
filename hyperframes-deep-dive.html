01 · The premise

What HyperFrames actually is

One sentence: open-source HTML→MP4 renderer with built-in browser NLE, agent skills, and deterministic seek across animation runtimes. Tagline: "Write HTML. Render video. Built for agents."

Built by HeyGen. Inspired by Remotion (they credit it in source) — same headless-Chrome + FFmpeg approach. The split: Remotion = React TSX. HF = HTML + data-* attributes.

Why HeyGen open-sourced it

Render engine isn't the moat. Avatars + lipsync + voice cloning are. HF is bait — community ships blocks and skills; HeyGen sells avatars on top.

License is Apache 2.0 (OSI-approved). Use commercially at any scale. No per-render fees. No seat caps. Compare Remotion: source-available, paid above small-team thresholds.

05 · Schema

Core timeline types

From packages/core/src/core.types.ts. Read this once and the whole system clicks.

type TimelineElementType = "video" | "image" | "text" | "audio" | "composition";
type Orientation = "16:9" | "9:16";

CANVAS_DIMENSIONS = {
  landscape: { width: 1920, height: 1080 },
  portrait:  { width: 1080, height: 1920 },
};

interface TimelineElementBase {
  id, type, name; startTime, duration, zIndex; x?, y?, scale?, opacity?;
}

interface TimelineMediaElement extends TimelineElementBase {
  src, mediaStartTime?, sourceDuration?, isAroll?, volume?, hasAudio?
}

interface TimelineTextElement extends TimelineElementBase {
  content, color?, fontFamily?, fontSize?, fontWeight?,
  textShadow?, textOutline?, textHighlight?, textHighlightColor?, ...
}

interface TimelineCompositionElement extends TimelineElementBase {
  src, compositionId, variableValues?: Record<string, string|number|boolean>
}

type CompositionVariableType = "string" | "number" | "color" | "boolean" | "enum";
Strategic insight

Composition variables = template slots. We can build "Reel templates" with named slots like {title}, {ctaColor}, {logoSrc}, then stamp out hundreds of variants without LLM-generating HTML. This is how Videfy keeps token cost near zero on the free tier.

  • No fetch() during render
  • No live tickers / breaking news

  • NLE handles timing, not layout
  • No animation curve editor
  • Desktop only (no mobile authoring)
  • No audio waveform editing (display only)
⚠️
Workflow gaps
  • No multi-user collab / real-time
  • No comments / approvals (vs Frame.io)
  • No version history / branches
  • No client review portal
  • Single-machine render (no Lambda)

  • No render farm queue
⚠️

  • No analytics on rendered videos
08 · Architectural insight

Determinism = byte-identical renders

HF's superpower. Same input → same output, every machine, every time. The mechanism is worth copying wholesale.

1
Paused timelines
GSAP / Anime / etc registered on window.__timelines with paused: true. Animations never play forward on their own.
2
Per-frame seek hooks
Engine calls timeline.seek(t) for each frame, then captures. The runtime is driven, not free-running.
3
Wall-clock banned
No Date.now, no requestAnimationFrame for animation logic. Lint enforces this.
4
Font data baked in
fontData.generated.ts ensures cross-machine glyph consistency.
5
Frame Adapter pattern
Every animation runtime exposes a seek(t) interface. New runtimes plug in without changes to the engine.
Why this matters for Videfy

Pipelines need determinism. Re-runs become idempotent. Caching works. A/B variants are cheap. Distributed render becomes possible (chunk frames, recombine).

12 · Snapshot pattern

Live-data workaround — fetch + freeze

"No fetch() during render" looks like a hard limit, but there's a clean escape. Snapshot data at render submit time, write to project, render reads it synchronously.

1
Pre-render hook
Videfy fetches API → writes data.json to HF project
2
HTML reads sync
Composition reads data.json via injected <script> at compile
3
Render is deterministic
Given the snapshot, the output is byte-identical
4
Re-render = re-snapshot
Idempotent — newer data, fresh render, no fragility

Better still: use HF composition variableValues. Already supported. Videfy fills variables, HF stamps the variant.

14 · The competition

How Videfy + Claude beat HeyGen

HeyGen — strengths + weaknesses

Strengths. 100+ stock avatars, custom avatar option, voice cloning, lip-sync, translation, dubbing, $89/mo Creator.

Weaknesses.

  • Avatars look uncanny corporate — nobody scrolls past one on TikTok
  • UI clicker, not agent — no "make 50 reel variants" workflow
  • Locked to their avatars, no NB2 / MJ / Veo / Sora
  • No real-footage + AI mix
  • B2B explainer focus, not reels / TikTok / IG
  • Per-seat pricing kills agencies + multi-account creators
  • API exists but locked-down + expensive
Videfy — what makes it win
  • Agent-native: script → assets → render → post in one prompt
  • Multi-engine gen: NB2, MJ, ChatGPT, Veo, Seedance, Kling, Sora
  • Real footage + AI mix — Pexels B-roll + AI gen + user assets
  • Editorial look — film stock, grain, no corporate avatar tells
  • Reels / TikTok native — 9:16, hook-first, scroll-stoppers
  • Pipeline composability — chain steps, schedule, batch
  • Open render layer (HF Apache 2.0), no per-render fees
  • BYOK pricing — bring your own keys, pay actual cost
  • Voice clone + lip-sync via OSS (ElevenLabs + Wav2Lip / LatentSync)
  • Skills marketplace: /videfy ecosystem
  • Multi-account creator + agency support — one workspace, N brands
  • Determinism enables overnight 100-variant batches
Positioning

"HeyGen for creators, not corporates."

Tagline: Type your reel. Get a reel. No avatars. ICP: creators 5K-100K followers, agencies running 5+ brands, content shops. Pricing: $29/mo Creator (BYOK), $99/mo Studio (managed gen + cloud render), $399/mo Agency (10 brands).

What HeyGen can't easily copy

Open-source positioning — their model = closed gen + locked avatars. Multi-engine gen — economics depend on selling their avatars. Editorial aesthetic — brand identity is corporate-explainer. Agent-native workflows — UI is template-clicker; rebuild = a new product.
