01 · Get a working dev env

Dev setup — 5 commands

From zero to working Videfy + HF dev environment.

# 1. Prereqs (Mac)
brew install node ffmpeg git-lfs && git lfs install

# 2. Clone HF source (for reading + reference, not editing)
git clone --depth 1 https://github.com/heygen-com/hyperframes.git ~/dev/hyperframes

# 3. Scaffold a Videfy test project (HF as render backend)
npx hyperframes init ~/dev/videfy-test --tailwind
cd ~/dev/videfy-test && npm install

# 4. Install HF skills globally (slash commands in Claude Code, Cursor, Codex)
npx skills add heygen-com/hyperframes --yes --global

# 5. Boot Studio (browser editor at localhost:3002)
npm run dev
Verify

Open http://localhost:3002 — you should see the HF Studio with NLE timeline + assets sidebar + render queue. Drop a PNG into ~/dev/videfy-test/assets/ and it should appear in the Assets tab automatically.

02 · Real skill file

Skill skeleton: /videfy-gen

Lives at ~/.agents/skills/videfy-gen/SKILL.md. Frontmatter declares trigger criteria. Markdown body becomes system prompt prepended to user requests. Tool calls inherit full agent permissions.

~/.agents/skills/videfy-gen/SKILL.md 
---
name: videfy-gen
description: Generate images and short video clips for Videfy compositions. Use when the user asks to generate, create, or stage AI images or AI video clips for inclusion in a HyperFrames composition. Supports NanoBanana 2 (image), Midjourney (image), ChatGPT (image), Veo 3.1 (video), Kling (video), Seedance (video). Always writes outputs to the active HyperFrames project's `assets/` directory so HF Studio picks them up automatically.
---

# /videfy-gen

You generate AI imagery and short video clips for Videfy compositions.
Outputs MUST land in the active HF project's `assets/` directory.

## Resolution rules

1. Resolve the active HF project path from the conversation context.
   - User explicit path? Use it.
   - User in a `npx hyperframes preview` session? Read its project from `localhost:3002/projects`.
   - Otherwise ASK: "Which HF project should I write into?"

2. Aspect ratio defaults:
   - 9:16 portrait for reels / TikTok / Stories
   - 16:9 landscape for YouTube / explainers / web heroes
   - 1:1 for IG feed posts

3. Engine selection (default order):
   - Photorealistic editorial → ChatGPT (`gpt-image-2`) or NB2
   - Surreal / cinematic → Midjourney
   - Motion clip → Veo 3.1 (start frame from NB2 image)

## Tool usage

Always call `videfy-daemon` first via HTTP:

```
POST http://localhost:7777/generate/image
  { "prompt": "...", "engine": "nb2", "aspect": "9:16",
    "out": "<hf-project>/assets/<slug>-<n>.png", "n": 4 }
```

The daemon downloads the image bytes and writes to `out`. HF Studio
auto-refreshes the Assets tab via file watcher.

## Banned engines

NEVER use Flux. Stick to NB2, NB Pro, Midjourney, ChatGPT, Veo, Kling.

## Realism rules

- Film stock, grain, imperfection language
- No "beautiful" / "stunning" / "perfect" / "8K" / "hyperrealistic"
- No forced logos
- Always humans in frame (no empty still lifes)

## After generation

- Verify file exists on disk
- Report path back to user
- Suggest the next step: "Drag this from Assets tab to track 1 at t=0:00"
- Do NOT auto-edit the composition unless user requests

## When to escalate

If user asks for a full reel: hand off to `/videfy` orchestrator.
If user wants to wire into composition HTML: hand off to `/hyperframes`.

Mirror this structure for /videfy-script, /videfy-post, /videfy-stock, /videfy-clip, /videfy-shotlist. Each ~50-100 lines, frontmatter + body + tool usage + banned-paths + escalation.

03 · Daemon code

Videfy daemon — Hono server

Local-first orchestrator on localhost:7777. Receives gen requests from skills, writes outputs to the active HF project, calls third-party gen APIs (Fal, Kie, OpenAI), returns absolute paths back. Stateless, deterministic, BYOK.

videfy-daemon/src/server.ts 
import { Hono } from "hono";
import { cors } from "hono/cors";
import { z } from "zod";
import { writeFile, mkdir } from "node:fs/promises";
import { dirname } from "node:path";
import { generateImage } from "./engines/index.ts";
import { generateVideo } from "./engines/video.ts";
import { synthVoice } from "./tts/kokoro.ts";
import { transcribe } from "./asr/whisper.ts";
import { searchBroll } from "./broll/pexels.ts";
import { postBlotato } from "./post/blotato.ts";

const app = new Hono();
app.use("*", cors({ origin: ["http://localhost:3002"] }));

const ImageReq = z.object({
  prompt: z.string().min(3),
  engine: z.enum(["nb2", "nb-pro", "mj", "chatgpt"]),
  aspect: z.enum(["9:16", "16:9", "1:1"]).default("9:16"),
  out: z.string(),       // absolute path inside HF project /assets
  n: z.number().int().min(1).max(8).default(1),
  refImages: z.array(z.string()).optional(),
});

app.post("/generate/image", async (c) => {
  const body = ImageReq.parse(await c.req.json());
  await mkdir(dirname(body.out), { recursive: true });
  const results = await generateImage(body);    // returns Buffer[]
  const paths: string[] = [];
  for (const [i, buf] of results.entries()) {
    const p = body.out.replace(/(\.\w+)$/, `_${i + 1}$1`);
    await writeFile(p, buf);
    paths.push(p);
  }
  return c.json({ ok: true, paths });
});

const VideoReq = z.object({
  prompt: z.string(),
  engine: z.enum(["veo-3.1", "kling-2.1", "seedance"]),
  startFrame: z.string().optional(),  // path to NB2 image
  duration: z.number().min(1).max(10).default(8),
  out: z.string(),
});
app.post("/generate/video", async (c) => {
  const body = VideoReq.parse(await c.req.json());
  await mkdir(dirname(body.out), { recursive: true });
  const buf = await generateVideo(body);
  await writeFile(body.out, buf);
  return c.json({ ok: true, path: body.out });
});

const TtsReq = z.object({
  text: z.string(),
  voice: z.string().default("af_heart"),
  out: z.string(),
});
app.post("/tts", async (c) => {
  const body = TtsReq.parse(await c.req.json());
  await synthVoice(body);
  return c.json({ ok: true, path: body.out });
});

const TranscribeReq = z.object({
  audioPath: z.string(),
  out: z.string(),  // writes word-level JSON
});
app.post("/transcribe", async (c) => {
  const body = TranscribeReq.parse(await c.req.json());
  await transcribe(body);
  return c.json({ ok: true, path: body.out });
});

app.post("/broll/search", async (c) => {
  const { query, source = "pexels", count = 10 } = await c.req.json();
  const results = await searchBroll({ query, source, count });
  return c.json({ ok: true, results });
});

const PostReq = z.object({
  videoPath: z.string(),
  caption: z.string(),
  accountIds: z.array(z.string()),
  scheduledTime: z.string(),  // ISO 8601
});
app.post("/post/blotato", async (c) => {
  const body = PostReq.parse(await c.req.json());
  const submissionId = await postBlotato(body);
  return c.json({ ok: true, submissionId });
});

// Health check
app.get("/healthz", (c) => c.json({ ok: true, version: "0.1.0" }));

export default { port: 7777, fetch: app.fetch };
videfy-daemon/src/engines/index.ts — engine router 
import { generateNB2 } from "./nb2.ts";
import { generateMJ } from "./mj.ts";
import { generateChatGPT } from "./chatgpt.ts";

export async function generateImage(req: {
  prompt: string;
  engine: "nb2" | "nb-pro" | "mj" | "chatgpt";
  aspect: "9:16" | "16:9" | "1:1";
  n: number;
  refImages?: string[];
}): Promise<Buffer[]> {
  switch (req.engine) {
    case "nb2":
    case "nb-pro":  return generateNB2(req);
    case "mj":      return generateMJ(req);
    case "chatgpt": return generateChatGPT(req);
  }
}
Why a daemon vs library

Daemon = language-agnostic. Skills call HTTP, not Node imports. Future Videfy clients (Python ML pipelines, n8n nodes, hosted web app) hit the same routes. Plus skills run inside Claude Code's process — calling external tools needs a network boundary anyway.

04 · API recipes

Studio API — every call as a curl

Assumes Studio running at localhost:3002. Replace :id with your HF project ID (slug from meta.json).

List projects 
curl -s http://localhost:3002/api/projects | jq
Read a file (e.g. composition HTML) 
curl -s http://localhost:3002/api/projects/my-video/files/index.html
Write a file (overwrite) 
curl -s -X PUT http://localhost:3002/api/projects/my-video/files/index.html \
  -H "Content-Type: text/plain" \
  --data-binary @local-index.html
Patch a file (partial update — JSON patch / text edit) 
curl -s -X PATCH http://localhost:3002/api/projects/my-video/files/index.html \
  -H "Content-Type: application/json" \
  -d '{ "find": "data-duration=\"5\"", "replace": "data-duration=\"7\"" }'
Drop an asset (e.g. generated image) 
curl -s -X POST http://localhost:3002/api/projects/my-video/files/assets/remi-001.png \
  --data-binary @/path/to/remi-001.png
Lint composition 
curl -s http://localhost:3002/api/projects/my-video/lint | jq
Get frame thumbnail at t=2.5s, 480px wide 
curl -s "http://localhost:3002/api/projects/my-video/thumbnail/index.html?t=2.5&w=480&format=jpeg" \
  -o thumb.jpg
Get audio waveform peaks 
curl -s http://localhost:3002/api/projects/my-video/waveform/assets/voiceover.wav | jq
Submit render 
curl -s -X POST http://localhost:3002/api/projects/my-video/render \
  -H "Content-Type: application/json" \
  -d '{
    "compositionPath": "index.html",
    "format": "mp4",
    "fps": 30,
    "width": 1080,
    "height": 1920,
    "quality": "high"
  }' | jq
# returns { jobId: "abc123", status: "queued" }
Poll render progress 
curl -s http://localhost:3002/api/render/abc123/progress | jq
# returns { status: "rendering", progress: 0.42, eta: 38 }
Download finished MP4 
curl -s http://localhost:3002/api/render/abc123/download -o output.mp4
Cancel render 
curl -s -X DELETE http://localhost:3002/api/render/abc123
Routing prefix

Studio API mounts under /api by default. Verify with curl http://localhost:3002/api/projects on your install — older versions used root paths. The handler factory is at packages/core/src/studio-api/createStudioApi.ts.

07 · Cloud render

Dockerfile.render — Modal / Fly.io

Headless HF renderer in a container. Submit jobs from Videfy daemon, rendered MP4 piped back. Base on HF's own Dockerfile.test — known-working Chrome + ffmpeg combo.

FROM node:22-bookworm-slim

# System deps for headless Chrome + ffmpeg (HDR-capable build)
RUN apt-get update && apt-get install -y --no-install-recommends \
      ffmpeg ca-certificates fonts-liberation libnss3 libatk1.0-0 \
      libatk-bridge2.0-0 libcups2 libdrm2 libxkbcommon0 libxcomposite1 \
      libxdamage1 libxfixes3 libxrandr2 libgbm1 libpango-1.0-0 \
      libcairo2 libasound2 libxshmfence1 wget gnupg \
  && rm -rf /var/lib/apt/lists/*

# Chrome (Puppeteer's bundled Chromium ships during npm install)
ENV PUPPETEER_DOWNLOAD_PATH=/opt/chromium
ENV NODE_ENV=production

WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci --omit=dev

COPY . .

# Render worker entrypoint — receives jobs via HTTP POST, runs HF render,
# returns MP4 path. Internally uses @hyperframes/producer.
EXPOSE 8080
CMD ["node", "dist/render-worker.js"]
Modal deploy snippet 
import modal

app = modal.App("videfy-render")
image = (modal.Image.from_dockerfile("Dockerfile.render")
         .pip_install("modal")
         .add_local_dir("./", remote_path="/app"))

@app.function(image=image, gpu=None, cpu=4, memory=8192, timeout=600)
def render(job: dict) -> bytes:
    # Job: { compositionPath, variables, fps, width, height, quality }
    import subprocess, json
    out = "/tmp/output.mp4"
    subprocess.run(
        ["node", "dist/render-worker.js", "--job", json.dumps(job), "--out", out],
        check=True,
    )
    with open(out, "rb") as f:
        return f.read()

09 · Token economics

Prompt cache strategy

Anthropic's prompt cache = 90% discount on repeated prefix tokens, 5-minute TTL on standard cache, 1-hour TTL on extended cache. Skills are big system prompts — caching them is the single biggest cost lever.

Rules

  1. Stable prefix. Skill bodies don't change between calls. Make them the first system block, mark cache_control: { type: "ephemeral" } on the last item of the cacheable section.
  2. Variables go after the cache cut. User prompt + composition state = post-cache. The cache key is the prefix hash.
  3. Keep skills under 2K tokens each. Bigger = pricier cold start. Move long examples into references/ and load on demand via tool call, not prefix.
  4. Use extended cache for the orchestrator. /videfy runs across long sessions — 1-hour TTL is worth the slight surcharge.
  5. Watch the metric. Token usage event includes cache_creation_input_tokens and cache_read_input_tokens. Read = 90% off. Creation = full price (one-time). Aim for cache_read > 95% of repeat calls within TTL.

Anthropic SDK snippet

import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic();
const skillPrompt = await readFile(
  "~/.agents/skills/videfy/SKILL.md", "utf8");

const resp = await client.messages.create({
  model: "claude-sonnet-4-6",
  max_tokens: 1024,
  system: [
    {
      type: "text",
      text: skillPrompt,
      cache_control: { type: "ephemeral" }   // 5-min TTL
    }
  ],
  messages: [
    { role: "user", content: userMessage }
  ]
});
Free-tier math

Skill prompt ~1500 tokens. Without cache: 1500 × $3/MTok input = $0.0045 per call. With cache after warm-up: 1500 × $0.30/MTok = $0.00045 per call. 10× saving. 100 calls/day per user × 1000 users × 30 days = $1,350/mo without cache, $135 with. Easy decision.
